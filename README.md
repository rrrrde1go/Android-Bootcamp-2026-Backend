  # Техническое задание на разработку программного продукта для планирования встреч
  
  ## 1. Общие описание
  Необходимо разработать программный продукт для планирования встреч сотрудников внутри компании. Система состоит из клиентского мобильного приложения и серверной части.
  
  ## 2. Клиентская часть (Android-приложение)
  Клиентом является **Android-приложение**, которое должно быть реализовано на **Kotlin** или **Java** с использованием **адаптивной верстки**.
  
  Приложение должно предоставлять интуитивно понятный интерфейс, в котором реализован следующий функционал:
  
  *   **Регистрация и авторизация** сотрудника.
  *   **Настройка личного профиля**.
  *   **Создание встречи** с возможностью приглашения других сотрудников. Приглашенные сотрудники должны иметь возможность дать бинарный ответ (**принял** / **не принял**).
  *   **Просмотр списка активных приглашений** на встречи с возможностью дать ответ о возможности посещения.
  *   **Просмотр собственного расписания встреч** в разрезе дня, недели и месяца.
  
  **Важное условие:** временные слоты для встреч разбиты строго по часам. Например, можно создать встречу на 9:00–10:00, но создание встречи на 9:10, 9:05 и т.д. — невозможно.
  
  ## 3. Серверная часть
  Сервер представляет собой **микросервис на Spring Boot**, работающий с СУБД **PostgreSQL** или **H2**.
  
  Серверное приложение должно:
  *   Предоставлять API, необходимый для полноценной работы мобильного приложения
  *   Осуществлять все основные операции с данными в базе через **Spring Data JPA**
  *   Использовать **Liquibase** для создания схемы базы данных и ее предзаполнения
  
  ## 4. User Stories
  * Как сотрудник, я хочу зарегистрироваться в системе, чтобы иметь собственную учётную запись
  * Как сотрудник, я хочу авторизоваться в системе, чтобы получить доступ к основному функционалу приложения
  * Как сотрудник, я хочу редактировать личную информацию в своём профиле, чтобы данные были актуальными
  * Как сотрудник, я хочу создавать встречи, указав дату, время и её участников, чтобы организовать встречу нужных сотрудников в нужное время
  * Как сотрудник, я хочу иметь возможность как отклонить, так и принять приглашения от других сотрудников, чтобы самому выбирать нужные мне встречи
  * Как сотрудник, я хочу видеть список активных приглашений, чтобы эффективно давать ответы о возможности посещения
  * Как сотрудник, я хочу просматривать собственное расписание в разрезе дня, недели и месяца, чтобы не пропустить важные встречи
  
  ## 5. Use Cases
1. ### Регистрация нового сотрудника в системе
  * **Цель**: Создание новой учётной записи
  * **Участники**: Сотрудник
  * **Предусловия**: У сотрудника ещё нет собственной учётной записи
  * **Основной поток**:
      * Сотрудник вводит ФИО, электронную почту и придумывает пароль
      * Приложение отправляет запрос на сервер
      * Приложение переходит на экран авторизации
  * **Альтернативный поток**:
      * Электронная почта уже используется для другой учётной записи
      * На сервере отсутствует информация о сотруднике, в доступе отказать
      * Соответствующая ошибка выводится на экран
2. ### Авторизация сотрудника
  * **Цель**: Получения сотрудником доступа к основным функция приложения
  * **Участники**: Сотрудник
  * **Предусловия**: Сотрудник уже создал учётную запись
  * **Основной поток**:
    * Сотрудник вводит электронную почту и пароль от своей учётной записи
    * Приложение отправляет запрос на сервер
    * Переход на главный экран
  * **Альтернативный поток**:
    * Неверная электронная почта или пароль
    * Сообщение об ошибке выводится на экран
3. ### Настройка профиля
  * **Цель**: Поддержание личной информации в профиле сотрудника актуальной
  * **Участники**: Сотрудник 
  * **Предусловия**: Сотрудник авторизован
  * **Основной поток**:
    * Сотрудник обновляет фотографию профиля, сделав снимок камерой или выбрать фотографию из галереи
    * Сотрудник изменяет ФИО
  * **Альтернативный поток**:
    * Нет подключения к интернету
    * Ошибка выводится на экран
4. ### Создание встречи
  * **Цель**: Организация встречи сотрудников
  * **Участники**: Сотрудник
  * **Предусловия**: Сотрудник авторизован
  * **Основной поток**: 
    * Сотрудник выбирает дату, час проведения встречи, а также сотрудников, которых хочет пригласить на встречу
    * Сотрудник подтверждает создание встречи, отправляется запросн на сервер
    * Указанным сотрудникам приходит приглашение, на которое они могут дать бинарный ответ
    * У принявших приглашение сотрудников встреча появляется в расписании
  * **Альтернативный поток**:
    * Встреча на указанное время уже была создана другим сотрудником, ошибка выводится на экран
    * Все указанные сотрудники отклонили приглашение, встреча отменена
5. ### Ответ на приглашение
  * **Цель**: Дать возможность сотруднику как принять, так и отклонить приглашение
  * **Участники**: Сотрудник
  * **Предусловия**: Сотрудник авторизован и был приглашен на встречу другим сотрудником
  * **Основной поток**:
    * Сотрудник принимает приглашение
    * Встреча появляется у него в расписании
  * **Альтернативный поток**:
    * У сотрудника уже запланирована встреча на данное время
    * Сотрудник отклоняет приглашение
6. ### Просмотр расписания
  * **Цель**: Увидеть расписание встреч в разрезе дня / недели / месяца
  * **Участники**: Сотрудник
  * **Предусловия**: Сотрудник авторизован
  * **Основной поток**:
    * Сотрудник переходит на экран с расписанием
    * Сотрудник выбирает временной промежуток, в котором хочет увидеть встречи
  * **Альтернативный поток**:
    * Нет подключения к интернету, на экран выводится предупреждение

  ## 6. Задачи
1. ### Разработать API для регистрации и авторизации сотрудников
  * **Реализовать эндпоинты для регистрации и входа в систему**
  * **Использовать JWT для управления авторизацией**
2. ### Настроить разграничение доступа
  * **Настроить Spring Security**
  * **Ограничить доступ к защищённым ресурсам только для авторизованных пользователей**
3. ### Разработать API для работы с профилем сотрудника
  * **Реализовать получение и редактирование данных профиля текущего пользователя**
4. ### Разработать API для получения списка сотрудников
  * **Реализовать эндпоинт получения списка сотрудников**
  * **Поддержать фильтрацию по ФИО**
5. ### Разработать API для создания и управления встречами
  * **Реализовать создание встреч с указанием даты, времени и участников**
  * **Обеспечить создание встреч только в часовых временных слотах**
6. ### Разработать API для обработки приглашений на встречи
  * **Реализовать получение списка приглашений**
  * **Обеспечить возможность принять или отклонить приглашение**
7. ### Реализовать работу с базой данных
  * **Использовать Spring Data JPA**
  * **Настроить миграции базы данных с помощью Liquibase**
